# Standalone Implementation Options

Updated On 27 March 2026 |
 2 Min(s) read

Feedback
Summarize

This section details about the implementation options available when the Temenos Payments Hub is deployed in Standalone mode. Temenos Payments Hub provides various implementation options for clients who can have different to-be states based on their business needs. Every option (model) describes about the:

- Integration touch points for inward and outward Credit Transfer (CT) and Direct Debit (DD).
- Information on who is responsible to do the integration work between the systems.
- Type of the integration required like Rest API, IF Event and Message and so on.
- Integration to an exit point which is mandatory or optional.

In TPH Standalone payments instance, for payments initiated and repaired from order entry, the replication of the account and customer information is mandatory.

There are different models as implementation options available for the clients in Standalone.
